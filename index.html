Complete React.js Mastery Guide

Learn React from absolute beginner to professional level with this comprehensive guide covering everything from fundamentals to advanced patterns.

What is React?

React is a JavaScript library for building user interfaces, created by Facebook. It follows a component-based architecture and uses a virtual DOM for efficient updates.

Why Learn React?

  • Most popular frontend library
  • High demand in job market
  • Reusable components
  • Great performance with Virtual DOM
  • Strong community and ecosystem
Prerequisites

Before learning React, you should be comfortable with:

  • HTML & CSS fundamentals
  • JavaScript (ES6+) including:
    • Arrow functions
    • Destructuring
    • Spread/Rest operators
    • Promises & Async/Await
    • Modules (import/export)
  • Basic understanding of npm/yarn

Core Fundamentals

1. JSX (JavaScript XML)

JSX is a syntax extension that allows writing HTML-like code in JavaScript.

// JSX Example
const element = <h1>Hello, world!</h1>;

// Embedding Expressions
const name = 'John';
const element = <h1>Hello, {name}</h1>;

// JSX is an expression too
function getGreeting(user) {
  if (user) {
    return <h1>Hello, {user.name}!</h1>;
  }
  return <h1>Hello, Stranger.</h1>;
}
Important JSX Rules
  • Use className instead of class
  • Use htmlFor instead of for
  • Always close tags ( <br/>, <img/> )
  • Use camelCase for attributes (onClick, tabIndex)
  • Only one root element per component
  • Use fragments (<>...) for multiple elements

2. Components

Components are reusable, independent pieces of UI. Two types:

Function Components

function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

// Arrow function style
const Welcome = (props) => {
  return <h1>Hello, {props.name}</h1>;
};

Class Components

class Welcome extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}</h1>;
  }
}
Best Practice

Use function components with hooks for new projects. Class components are mostly used in legacy code.

3. Props (Properties)

Props are read-only data passed from parent to child components.

// Parent Component
function App() {
  return (
    <Welcome name="John" age={25} />
  );
}

// Child Component
function Welcome(props) {
  return (
    <div>
      <h1>Hello, {props.name}</h1>
      <p>Age: {props.age}</p>
    </div>
  );
}

// Destructuring props
function Welcome({ name, age }) {
  return <h1>Hello, {name}</h1>;
}

4. State

State is mutable data that belongs to a component and triggers re-renders when changed.

// Class Component State
class Counter extends React.Component {
  constructor(props) {
    super(props);
    this.state = {
      count: 0
    };
  }

  increment = () => {
    this.setState({ count: this.state.count + 1 });
  }

  render() {
    return (
      <div>
        <p>Count: {this.state.count}</p>
        <button onClick={this.increment}>Increment</button>
      </div>
    );
  }
}
State Rules
  • Never modify state directly (use setState or useState setter)
  • State updates are asynchronous
  • State updates are merged (class components) or replaced (function components)
  • Lift state up to the closest common ancestor when sharing between components

React Hooks

Hooks let you use state and other React features in function components.

1. useState Hook Basic

Adds state to function components.

import React, { useState } from 'react';

function Counter() {
  // Declare state variable
  const [count, setCount] = useState(0);
  const [user, setUser] = useState({ name: '', age: 0 });

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>

      <!-- Update object state -->
      <button onClick={() => setUser({...user, age: user.age + 1})}>
        Increase Age
      </button>
    </div>
  );
}
useState Tips
  • Use functional updates when new state depends on previous state: setCount(prev => prev + 1)
  • Initialize with a function for expensive computations: useState(() => expensiveComputation())
  • Multiple state variables are fine - don't feel forced to combine them

2. useEffect Hook Essential

Handles side effects in function components.

import React, { useState, useEffect } from 'react';

function Example() {
  const [count, setCount] = useState(0);

  // Similar to componentDidMount + componentDidUpdate
  useEffect(() => {
    // Update document title
    document.title = `You clicked ${count} times`;
  });

  // Runs only on mount (empty dependency array)
  useEffect(() => {
    console.log('Component mounted');
    
    // Cleanup function (runs on unmount)
    return () => {
      console.log('Component will unmount');
    };
  }, []);

  // Runs when count changes
  useEffect(() => {
    console.log(`Count changed to: ${count}`);
  }, [count]);

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </div>
  );
}
useEffect Best Practices
  • Always specify dependency array
  • Clean up subscriptions, timers, and event listeners
  • Separate concerns into multiple effects
  • Use useEffect for side effects, not for data transformation
  • Consider using useLayoutEffect for DOM measurements

3. useContext Hook Advanced

Access context values without nesting.

// 1. Create Context
const ThemeContext = React.createContext('light');

// 2. Provide Context
function App() {
  return (
    <ThemeContext.Provider value="dark">
      <Toolbar />
    </ThemeContext.Provider>

}

// 3. Consume Context
function Toolbar() {
  const theme = useContext(ThemeContext);
  return <div className={`button-${theme}`}>Theme: {theme}</div>;
}

4. useRef Hook Essential

Creates a mutable reference that persists across renders.

import React, { useRef, useEffect } from 'react';

function TextInput() {
  // Create ref
  const inputRef = useRef(null);
  const renderCount = useRef(1);

  useEffect(() => {
    // Focus input on mount
    inputRef.current.focus();
    
    // Track render count
    renderCount.current += 1;
  });

  return (
    <div>
      <input ref={inputRef} type="text" />
      <p>Renders: {renderCount.current}</p>
    </div>
  );
}
useRef vs useState

useRef: Changes don't trigger re-renders. Good for DOM references, timers, previous values.

useState: Changes trigger re-renders. Good for data that affects UI.

5. useReducer Hook Advanced

Alternative to useState for complex state logic.

const initialState = { count: 0 };

function reducer(state, action) {
  switch (action.type) {
    case 'increment':
      return { count: state.count + 1 };
    case 'decrement':
      return { count: state.count - 1 };
    case 'reset':
      return initialState;
    default:
      throw new Error();
  }
}

function Counter() {
  const [state, dispatch] = useReducer(reducer, initialState);

  return (
    <div>
      Count: {state.count}
      <button onClick={() => dispatch({ type: 'increment' })}>+</button>
      <button onClick={() => dispatch({ type: 'decrement' })}>-</button>
      <button onClick={() => dispatch({ type: 'reset' })}>Reset</button>
    </div>
  );
}

6. Custom Hooks Advanced

Extract component logic into reusable functions.

// Custom hook: useLocalStorage
function useLocalStorage(key, initialValue) {
  const [storedValue, setStoredValue] = useState(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      return initialValue;
    }
  });

  const setValue = (value) => {
    try {
      const valueToStore =
        value instanceof Function ? value(storedValue) : value;
      setStoredValue(valueToStore);
      window.localStorage.setItem(key, JSON.stringify(valueToStore));
    } catch (error) {
      console.log(error);
    }
  };

  return [storedValue, setValue];
}

// Using the custom hook
function Component() {
  const [name, setName] = useLocalStorage('name', 'John');

  return (
    <input
      value={name}
      onChange={e => setName(e.target.value)}
    />
  );
}

Advanced Concepts

1. Higher-Order Components (HOC)

A pattern where a function takes a component and returns a new component.

// HOC: withAuthentication
const withAuthentication = (WrappedComponent) => {
  return function (props) {
    const [isAuthenticated, setIsAuthenticated] = useState(false);

    if (!isAuthenticated) {
      return <Login onLogin={() => setIsAuthenticated(true)} />;
    }

    return <WrappedComponent {...props} />;
  };
};

// Usage
const Dashboard = () => <div>Dashboard Content</div>;
const AuthenticatedDashboard = withAuthentication(Dashboard);

2. Render Props Pattern

A component prop that is a function returning React elements.

function MouseTracker(props) {
  const [position, setPosition] = useState({ x: 0, y: 0 });

  const handleMouseMove = (event) => {
    setPosition({
      x: event.clientX,
      y: event.clientY
    });
  };

  return (
    <div style={{ height: '100vh' }} onMouseMove={handleMouseMove}>
      {props.render(position)}
    </div>
  );
}

// Usage
<MouseTracker
  render={position => (
    <>
      <h1>Mouse position</h1>
      <p>X: {position.x}, Y: {position.y}</p>
    </>
  )}
/>

3. React.memo

Performance optimization - memoizes component to prevent unnecessary re-renders.

const ExpensiveComponent = React.memo(function ExpensiveComponent(props) {
  // Only re-renders if props change
  console.log('Expensive render');
  return <div>{props.value}</div>;
});

// Custom comparison function
const areEqual = (prevProps, nextProps) => {
  return prevProps.value === nextProps.value;
};

const CustomMemoComponent = React.memo(ExpensiveComponent, areEqual);

Load components only when needed to improve performance.

      <Suspense fallback={<div>Loading...</div>}>
        <LazyComponent />

    </div>

}

// Route-based code splitting (with React Router)
const Home = lazy(() => import('./routes/Home'));

5. Error Boundaries

React components that catch JavaScript errors anywhere in their child component tree.

class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false };
  }

  static getDerivedStateFromError(error) {
    return { hasError: true };
  }

  componentDidCatch(error, errorInfo) {
    // Log to error reporting service
    console.log('Error:', error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      return <h1>Something went wrong.</h1>;
    }

    return this.props.children;
  }
}

// Usage
<ErrorBoundary>
  <MyComponent />
</ErrorBoundary>

6. Portals

Render children into a DOM node that exists outside the parent component's DOM hierarchy.

function Modal({ children }) {
  const modalRoot = document.getElementById('modal-root');
  return ReactDOM.createPortal(
    <div className="modal">
      {children}
    </div>,
    modalRoot
  );
}

// HTML
<div id="root"></div>
<div id="modal-root"></div>

7. Forwarding Refs

Pass ref through a component to one of its children.

// Forward ref to input element
const FancyInput = React.forwardRef((props, ref) => (
  <input ref={ref} className="fancy-input" {...props} />
));

// Parent component
function App() {
  const inputRef = useRef(null);

  useEffect(() => {
    inputRef.current.focus();
  }, []);

  return <FancyInput ref={inputRef} />;
}

State Management

1. Context API (Built-in)

For global state without external libraries.

// Create Context
const ThemeContext = React.createContext();

// Provider Component
function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');

  const toggleTheme = () => {
    setTheme(prev => prev === 'light' ? 'dark' : 'light');
  };

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

// Custom Hook
export function useTheme() {
  const context = useContext(ThemeContext);
  if (!context) {
    throw new Error('useTheme must be used within ThemeProvider');
  }
  return context;
}

// Usage
function App() {
  return (
    <ThemeProvider>
      <Header />
      <Main />
    </ThemeProvider>
  );
}

function Header() {
  const { theme, toggleTheme } = useTheme();
  return (
    <button onClick={toggleTheme}>
      Switch to {theme === 'light' ? 'Dark' : 'Light'} Mode
    </button>
  );
}

1. Fetch API (Built-in)

function Users() {

        }
        const data = await response.json();

      } finally {
        setLoading(false);
      }
    };

    fetchUsers();
  }, []);

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}

Custom API Hook

import { useState, useCallback } from 'react';

function useApi(apiFunction) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  const execute = useCallback(async (...args) => {
    try {
      setLoading(true);
      setError(null);
      const result = await apiFunction(...args);
      setData(result);
      return result;
    } catch (err) {
      setError(err.message || 'Something went wrong');
      throw err;
    } finally {
      setLoading(false);
    }
  }, [apiFunction]);

  return { data, loading, error, execute };
}

// Usage
const fetchUsers = () => api.get('/users');
const { data: users, loading, error, execute: fetch } = useApi(fetchUsers);

useEffect(() => {
  fetch();
  // eslint-disable-next-line react-hooks/exhaustive-deps
}, []);

Prevents unnecessary re-renders of functional components.

  console.log('Rendering expensive component');
  return <div>{data}</div>;
});

// Custom comparison function
const areEqual = (prevProps, nextProps) => {
  return prevProps.data.id === nextProps.data.id;
};

const CustomMemoComponent = React.memo(ExpensiveComponent, areEqual);

2. useMemo

Memoizes expensive calculations.

function ExpensiveComponent({ items, filter }) {
  // Only recalculates when items or filter changes
  const filteredItems = useMemo(() => {
    return items.filter(item =>
      item.name.toLowerCase().includes(filter.toLowerCase())
    );
  }, [items, filter]);

  return <List items={filteredItems} />;
}

3. useCallback

Memoizes callback functions to prevent unnecessary re-creations.

function ParentComponent() {
  const [count, setCount] = useState(0);

  // This function is re-created on every render
  const handleClick = () => {
    console.log('Clicked');
  };

  // This function is memoized
  const memoizedHandleClick = useCallback(() => {
    console.log('Memoized click');
  }, []);

  return (
    <ChildComponent onClick={memoizedHandleClick} />
  );
}

Performance Checklist
  • Use React.memo for pure components
  • Memoize expensive calculations with useMemo
  • Prevent function recreation with useCallback
  • Implement virtualization for long lists
  • Use code splitting and lazy loading
  • Optimize images and assets

  • Use production builds for deployment

Common Performance Pitfalls
  • Creating functions inside JSX (use useCallback)
  • Passing new objects/arrays as props (memoize them)
  • Not using keys in lists
  • Overusing Context (can cause unnecessary re-renders)
  • Not cleaning up effects (memory leaks)
  • Rendering too many components at once

Single Responsibility Principle

Each component should do one thing well.

Composition over Inheritance

Build complex UIs from simple components.

Props Design

3. State Management Strategy

Local State (useState)

For component-specific data (forms, toggles, counters)

Lifted State

Share state between sibling components via common parent

Context API

Global state that doesn't change frequently (theme, auth)

State Management Library

Complex global state (Redux, Zustand, MobX)

Server State

API data (React Query, SWR, RTK Query)
